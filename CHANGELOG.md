# Changelog

All notable changes to Vibes to Verified will be recorded here.

## 0.1.0 — Release Candidate

- Added the portable `vibes-to-verified` Agent Skill.
- Defined the V0–V4 evidence maturity scale.
- Separated substantive verdict from evidence maturity.
- Added bounded independent approach and refutation workflows.
- Added a machine-readable evidence-card schema and semantic validator.
- Bound promotion evidence to both the main claim and exact final artifact.
- Required structured controlled-test, independent-review, claim-disposition, blocker, natural-runtime, monitoring, recovery, and read-back fields.
- Enforced GREEN consistency while preserving independent verdict and maturity axes.
- Enforced V2/V3/V4 promotion prerequisites in both Draft 2020-12 schema and Python semantics.
- Required separate natural-runtime and outcome-read-back artifacts for V4.
- Added regression fixtures for claim linkage, artifact identity, review independence, GREEN blockers, `GREEN / V0`, `RED / V4`, and incomplete V4 proof.
- Expanded privacy scanning to every tracked text file plus PNG and video metadata.
- Sanitized GPT Image source metadata without changing pixel content.
- Replaced publication destinations with placeholders until public read-back.
- Reclassified the public PAPER private-V4 claim as `BLOCKED / V0` because raw primary receipts remain private.
- Replaced a synthetic overclaiming README example with a schema-valid `BLOCKED / V0` fixture.
- Added an explicit validation dependency file and CI cache binding.
- Added coding, research, and sanitized options paper-trading examples.
- Added package, media, link, privacy, and video contract tests.
- Added deterministic SVG/PNG graphics and a GPT Image 2-based Article cover.
- Added a native 9:16 Manim launch video and source scene.
- Added the complete draft X Article and launch copy.

The release remains a candidate until independent review is resolved and the exact final artifacts are committed.
